The Participants tab manages the students attending an excursion. Use it to add students, track participation status, review parent relationships, and send parent permission emails when the required forms are available.

## What You Can See

- A Students heading with a count of participants.
- A note explaining whether you can manage participants or only view them.
- A message when no published permission form is available for parent permission emails.
- Actions for sending parent permission emails and adding students.
- A student membership grid with search, filters, editable cells, and context actions.
- Empty states for no students or no matching filter results.

## Main Actions

### Add Students

Use Add Students to open a transfer-style modal. The modal lets you move students from the available list into the excursion participant list, then save the changes.

### Send Parent Permission Emails

Use this action to send permission emails to families when the excursion has the right published permission form connected. If the action is unavailable, check the Forms tab for the required form setup.

### Email Selection

When you select relevant cells in the grid, the context menu can help you email selected students or parent contacts.

## Student Grid

The grid is designed for managing participant information quickly.

Common columns include:

- Student, showing the participant name and identifying details.
- Status, which can be updated and may show status history.
- Description, for participant-specific notes.
- Parents, showing linked parent or carer contacts.
- Permission email columns, showing permission send details and response status where available.

## Filters and Search

Use the toolbar to search and filter participants by student, parent, status, description, and other visible columns. If no records match, the page shows a filtered empty state.

## Common Workflows

### Build the participant list

Add students to the excursion, then review the grid to confirm the right students are included.

### Track permission progress

Use the permission-related columns to see which families have been contacted and what responses are available.

### Update a student's status

Change the participation status directly in the grid and review status history when you need more context.

## Tips

- Check parent relationships before sending permission emails.
- Use filters to find students who still need follow-up.
- Keep participant notes clear and relevant to the excursion.
