The Activity tab shows a timeline of updates for the excursion. Use it to understand what has changed, when it changed, and which records or actions may need follow-up.

## What You Can See

- A chronological activity feed for the excursion.
- Compact activity messages describing updates and changes.
- Links or references to related items when available.
- A properties column showing the excursion status, creator, creation date, and location.
- Loading, error, and empty states when activity is being loaded or no activity has been recorded yet.

## Main Sections

### Activity Feed

The feed records important updates connected to the excursion. Activity entries may relate to planning changes, linked items, comments, approvals, tasks, or other actions captured by the platform.

Each activity item is shown in a timeline style so staff can scan recent changes quickly.

### Empty Activity State

If there is no activity yet, the tab explains that updates will appear once people begin making changes to the excursion.

### Properties

The right column repeats important excursion properties. This lets you check the current status and location while reviewing the activity history.

## Common Workflows

### Catch up on recent changes

Open the Activity tab when you want to see what has happened since you last viewed the excursion.

### Investigate a planning change

Scan the timeline for recent entries, then open the related tab or item if you need more detail.

### Review context before approving

Check recent activity before approving or sending an excursion back for changes.

## Tips

- Use the Activity tab for history and context. Use the relevant feature tab to make changes.
- If the activity feed is empty, check other tabs for current excursion details.
- Review activity after major planning updates so you understand what changed.
