The Forms list page is where you browse workspace forms and open the form builder to edit fields, settings, and submissions.

## What this screen is for

- See all forms in the workspace.
- Create a new form shell before adding fields.
- Jump into the builder for an existing form.

## What you can see

- Page title **Forms** with the subtitle **Build reusable workspace forms and configure their fields.**
- A **New Form** button.
- A table with columns for **Title**, **Type**, **Fields**, **Updated**, and **Created**.
- Row actions including **Open builder**.
- Toolbar filters and a column visibility menu.
- A **Create Form** panel that collects a form title before creating the record.

## Common actions

### Create a form

Choose **New Form**, enter a **Title**, then choose **Create Form**. After creation, you are taken to the builder for that form.

If the title is missing, you see **Form title is required.**

### Open a form

Click a row to open the builder, or use the row menu action **Open builder**.

### Filter forms

Use the toolbar to search by title and filter by type. Use **Reset** when filters hide everything you expect to see.

### Adjust visible columns

Use the view options menu to show or hide columns, where available.

## Empty, loading, and error states

- While loading, the page shows a table skeleton.
- If the list cannot load, you see **Failed to load forms** with an error message.
- If there are no forms, you see **No forms yet.**
- If filters exclude all rows, you see **No forms match the current filters.**
- If you cannot create forms, the empty state explains you do not have permission to create forms.

## Permissions and read-only behaviour

**New Form** is disabled when you cannot manage forms. You may still be able to open existing forms depending on your access, but publishing and saving rules are enforced when you try to save in the builder.

## Tips

- Use short, recognisable titles; the title is used across lists and the builder header.
- Create the form shell first, then add fields in the builder.
- Use filters when a workspace has many forms.
