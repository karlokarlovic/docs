The Students page is the workspace roster for student records that can be linked to groups, excursions, imports, and family relationships.

## What this screen is for

- Browse student records.
- Create and edit students.
- Import many students from a spreadsheet.
- Open relationship management for a student.

## What you can see

- Page title **Students** with the subtitle **Create and maintain standalone student records that can be used elsewhere in the workspace.**
- **Import** and **New Student** actions.
- A table with columns for **Student**, **Date of birth**, **SIS ID**, **Student email**, and **Updated**.
- Toolbar filters and sorting.
- Row actions for **Edit student** and **Manage relationships**.

## Common actions

### Create a student

Choose **New Student** to open **Add Student** with the description **Add a student record that can be reused across the app later.**

### Edit a student

Choose **Edit student** from the row menu, or click the row when editing is allowed.

### Import students

Choose **Import** to open the import workflow.

### Manage relationships

Choose **Manage relationships** to open **Manage Relationships** for that student, with copy explaining you can create, edit, and end workspace-scoped person relationships.

## Student column behaviour

The **Student** column shows the primary display name. When a student has a preferred name and the displayed name differs from the legal name, a secondary line shows the legal name for clarity.

## Empty, loading, and error states

- If loading fails, you see **Failed to load students** with a message.
- If there are no students, you see **No students yet.**
- If filters hide all rows, you see **No students match the current filters.**

## Permissions and read-only behaviour

If you cannot manage students, **New Student** and **Import** are disabled, row click editing is disabled, and relationship management is not available.

## Tips

- Keep SIS identifiers consistent with your school source system.
- Use relationships to ensure parent communication targets the correct contacts.
- Use import for large onboarding batches instead of manual entry.
