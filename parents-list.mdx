The Parents page is the workspace directory for parent and carer records that can be linked to students and reused across the workspace.

## What this screen is for

- Browse parent records.
- Create and edit parents.
- Import many parents from a spreadsheet.
- Open relationship management for a parent.

## What you can see

- Page title **Parents** with the subtitle **Create and maintain standalone parent records that can be reused elsewhere in the workspace.**
- An **Import** link in the header actions area.
- A **New Parent** button.
- A table with columns for **Parent**, **Date of birth**, **SIS ID**, **Parent email**, and **Updated**.
- Toolbar filters and sorting.
- Row actions for **Edit parent** and **Manage relationships**.

## Common actions

### Create a parent

Choose **New Parent** to open **Add Parent** with the description **Add a parent record that can be reused across the app later.**

### Edit a parent

Choose **Edit parent** from the row menu, or click the row when editing is allowed.

### Import parents

Choose **Import** to open the import workflow.

### Manage relationships

Choose **Manage relationships** to open **Manage Relationships** for that parent, with copy explaining you can create, edit, and end workspace-scoped person relationships.

## Parent column behaviour

The **Parent** column shows the primary display name. When a preferred name exists and the displayed name differs from the legal name, a secondary line shows the legal name for clarity.

## Empty, loading, and error states

- If loading fails, you see **Failed to load parents** with a message.
- If there are no parents, you see **No parents yet.**
- If filters hide all rows, you see **No parents match the current filters.**

## Permissions and read-only behaviour

If you cannot manage parents, **New Parent** is disabled, row click editing is disabled, and relationship management is not available.

Note: the **Import** navigation control may still be visible even when you cannot import; if import is not permitted, the import page should surface read-only guidance and prevent successful imports.

## Tips

- Keep parent emails current before sending excursion communications.
- Use relationships to reflect guardianship and communication preferences accurately.
