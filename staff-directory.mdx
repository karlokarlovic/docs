The Staff directory lists everyone in the workspace so you can find colleagues, review membership details, and open a read-only excursion summary for a staff member.

## What this screen is for

- Browse workspace members.
- Filter the directory.
- Open a staff summary page for a selected member.

## What you can see

- Page title **Staff** with the subtitle **Browse all workspace users and open a read-only summary of each staff member's excursion history.**
- A table with columns such as **Staff member**, **Email**, **Workspace role**, **Membership status**, **Joined**, and **Compliance** summary.
- Toolbar filters and sorting.
- Clickable rows that navigate to the staff summary route.

## Common actions

### Open a staff member

Click a row to open the staff summary for that member.

### Filter and sort

Use filters to narrow down by role, status, compliance indicators, or text fields supported by the table.

## Empty, loading, and error states

- If loading fails, you see **Failed to load staff** with a message.
- If there are no members, you see **No staff found in this workspace.**
- If filters hide all rows, you see **No staff match the current filters.**

## Permissions and read-only behaviour

The directory is intended for browsing. Editing another user’s workspace role is handled elsewhere in workspace settings, not on this page.

## Tips

- Use the directory when assigning excursion staffing and you need to confirm who is in the workspace.
- Use filters when the workspace has many members.
